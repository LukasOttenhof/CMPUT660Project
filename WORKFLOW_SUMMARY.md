# sentiment_by_dataset.png Generation Workflow

## Complete Pipeline from Parquets to PNG

### Step 1: Data Collection & Parquet Generation
**Script:** `scripts/build_dataset/repo_analysis.py`
**Inputs:** GitHub repository URLs (from `filtered_repos_3year50pr.txt`)
**Outputs:** Parquet files in `inputs/50prs/`

Key parquet files generated:
- `commit_messages_before.parquet` / `commit_messages_after.parquet`
- `pr_bodies_before.parquet` / `pr_bodies_after.parquet`
- `issue_bodies_before.parquet` / `issue_bodies_after.parquet`
- `review_comments_before.parquet` / `review_comments_after.parquet`

Process:
1. Clones repositories from GitHub
2. Uses PyDriller to extract commits before/after boundary date
3. Uses GitHub API to fetch PRs, issues, and review comments
4. Splits data into "before" and "after" based on boundary date
5. Saves as parquet files for efficient storage

---

### Step 2: Topic Modeling (BERTopic)
**Script:** `scripts/analysis/analyze_rq4_topic_BERT.py`
**Dependencies:** 
  - `scripts/analysis/data_loader.py` (loads parquets)
  - `sentence-transformers/all-mpnet-base-v2` embedding model
  - BERTopic library
**Inputs:** Parquet files from Step 1
**Outputs:** Topic-related CSV files in `outputs/rq5/tables/`

Key output files:
- `rq5_doc_topics_commit_messages.csv`
- `rq5_doc_topics_pr_bodies.csv`
- `rq5_doc_topics_issue_bodies.csv`
- `rq5_doc_topics_review_comments.csv`

Process:
1. Loads parquets via `data_loader.load_all()`
2. Cleans text (removes AI-related terms, URLs, code blocks, etc.)
3. Segments long documents into sentences
4. Encodes using SentenceTransformers embeddings
5. Runs BERTopic to identify topics
6. Outputs doc-topic mappings with cleaned text and date info

---

### Step 3: Sentiment Analysis (VADER)
**Script:** `scripts/analysis/analyze_rq5_tone.py`
**Dependencies:** `vaderSentiment` library
**Inputs:** doc_topics CSV files from Step 2
**Outputs:** Tone CSV files (both overall and by-topic)

Output Files:
- `outputs/rq5/tables/tone_by_topic/rq5_tone_by_topic_commit_messages.csv`
- `outputs/rq5/tables/tone_by_topic/rq5_tone_by_topic_pr_bodies.csv`
- `outputs/rq5/tables/tone_by_topic/rq5_tone_by_topic_issue_bodies.csv`
- `outputs/rq5/tables/tone_by_topic/rq5_tone_by_topic_review_comments.csv`

Process:
1. Loads doc_topics CSV files (contains text and topic labels)
2. Applies VADER SentimentIntensityAnalyzer to each text item
3. Computes: neg, neu, pos, compound sentiment scores
4. Categorizes into: "positive" (>= 0.05), "neutral" (-0.05 to 0.05), "negative" (<= -0.05)
5. Groups by topic, group (before/after), and sentiment_cat
6. Calculates share (percentage) for each combination
7. Saves aggregated tone_by_topic CSV files

---

### Step 4: Generate the PNG Plot
**Script:** `scripts/analysis/topic_tone_plot.py`
**Dependencies:** matplotlib, pandas, numpy
**Inputs:** 

The script expects tone CSV files in `outputs/rq5/tables/`:
- `rq5_tone_commit_messages.csv`
- `rq5_tone_issue_bodies.csv`
- `rq5_tone_pr_bodies.csv`
- `rq5_tone_review_comments.csv`

**Issue:** These files are NOT created by analyze_rq5_tone.py. They need to be created separately!

**Output:**
- `outputs/rq5/plots/sentiment_by_dataset.png` ✅

Process in `plot_sentiment_per_dataset()`:
1. Reads tone CSV files for each dataset
2. Each has columns: text, date, time, group, neg, neu, pos, compound, sentiment_cat
3. Calculates share = count per (dataset, group, sentiment_cat) / total for that group
4. Calls `plot_stacked()` to create stacked bar chart
5. Two bars per dataset: "Before" and "After"
6. Stacked by sentiment: Negative (bottom) → Neutral (middle) → Positive (top)
7. Uses color scheme:
   - Before: Red (#E74C3C) with adjusted lightness for each sentiment
   - After: Blue (#3498DB) with adjusted lightness for each sentiment
8. Saves to PNG at 300 DPI

---

### Step 2.5: Generate Tone CSV Files (Direct from Parquets)
**Script:** `scripts/analysis/generate_tone_csv.py` (CREATED - was missing from original pipeline)
**Dependencies:** `vaderSentiment` library
**Inputs:** Parquet files from Step 1
**Outputs:** Tone CSV files in `outputs/rq5/tables/`

Key output files:
- `rq5_tone_commit_messages.csv`
- `rq5_tone_pr_bodies.csv`
- `rq5_tone_issue_bodies.csv`
- `rq5_tone_review_comments.csv`

Process:
1. Loads parquet files (before and after versions)
2. Combines before/after into single dataframe
3. Applies VADER SentimentIntensityAnalyzer to text
4. Computes: neg, neu, pos, compound sentiment scores
5. Categorizes into: "positive" (>= 0.05), "neutral" (-0.05 to 0.05), "negative" (<= -0.05)
6. Saves with all original text, date, and sentiment data

**Note:** This script was created to fill a gap in the pipeline. The tone files are required by topic_tone_plot.py but were not being generated by any identified script.

## Data Format Reference

### Input Parquet Format (from Step 1):
```
commit_messages_before.parquet columns:
- repo: string (repository name)
- author: string (commit author)
- date: datetime (commit date)
- source: string ("commit")
- text: string (commit message)
- sha: string (commit hash)
```

### Doc Topics Format (from Step 2):
```
rq5_doc_topics_commit_messages.csv columns:
- text: string (cleaned text)
- date: datetime
- group: string ("before" or "after")
- clean: string (preprocessed text without stopwords)
- topic_id: integer (BERTopic topic ID)
- topic_max_prob: float (confidence in topic assignment, if available)
```

### Tone Format (from Step 3 - by topic):
```
rq5_tone_by_topic_commit_messages.csv columns:
- topic_label: string (BERTopic topic label/name)
- group: string ("before" or "after")
- sentiment_cat: string ("positive", "neutral", or "negative")
- count: integer (number of documents)
- share: float (proportion within that topic-group combination)
```

### Tone Summary Format (for plotting):
```
rq5_tone_commit_messages.csv columns:
- text: string (original text)
- date: datetime
- time: datetime
- group: string ("before" or "after")
- neg: float (VADER negative score 0-1)
- neu: float (VADER neutral score 0-1)
- pos: float (VADER positive score 0-1)
- compound: float (VADER compound score -1 to 1)
- sentiment_cat: string ("positive", "neutral", or "negative")
```

## Understanding the PNG Output

### sentiment_by_dataset.png

The generated PNG shows:
- **X-axis:** Four datasets (Commit Messages, Issues, Pull Request Bodies, Review Comments)
- **Y-axis:** Share (0.0 to 1.0, representing 0-100%)
- **Bars:** Two bars per dataset (before on left, after on right)
- **Colors:**
  - **Red** shades for "before" period data
  - **Blue** shades for "after" period data
  - **Lighter shade** = positive sentiment
  - **Medium shade** = neutral sentiment
  - **Darker shade** = negative sentiment
- **Interpretation:** 
  - Taller lighter (positive) bars = more positive sentiment
  - Taller darker (negative) bars = more negative sentiment
  - Bars that shift left-to-right between before/after indicate sentiment change
